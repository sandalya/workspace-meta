# COMMANDS.md — Команди Кота

## Швидкі команди

| Команда   | Що робить                                                | Час  |
| --------- | -------------------------------------------------------- | ---- |
| `гіт`     | `git add -A` + commit + push в **активному проекті**     | 30с  |
| `гіткіт`  | `git add -A` + commit + push у workspace (мої файли)     | 30с  |
| `фікс`    | Зберегти успішне налаштування + документація + commit    | 45с  |
| `чкп`     | Memory flush + git обидва репо + звіт                    | 2хв  |

### гіт
```bash
cd [АКТИВНИЙ_ПРОЕКТ] && git add -A && git commit -m "feat: [опис]" && git push
```

### гіткіт
```bash
cd ~/.openclaw/workspace && git add -A && git commit -m "ai: [що змінилось]" && git push
```

### фікс
1. Backup конфігів (якщо треба)
2. Документація що виправлено
3. Git commit з описовим повідомленням

### чкп
1. Memory flush → `memory/YYYY-MM-DD.md` (технічний контекст)
2. Git commit + push обидва репо
3. Звіт:
```
🐱 ЧКП [дата]
✅ Зроблено: [список]
📁 Git: [commits]
🧠 Контекст збережено
🚀 Далі: [priorities]
```
Сесія продовжується (не робити `/new` автоматично).

## Правила розробки

### Перед restart — перевір дублікати
```bash
ps aux | grep main.py | grep -v grep
```
Якщо >1 процес → kill всі, потім запускай.

### "Готово" тільки після тесту
1. Перезапустити бота
2. Перевірити логи: `tail -20 [шлях_до_логів]`
3. Протестувати змінену функцію
4. Тільки тоді — звітувати

### Перед тестом бота
**Завжди** нагадай Сашку: *"Запусти логи ПЕРЕД тим як слати повідомлення боту."*

### Перемикання моделей
Команда: "переключись на haiku/sonnet/opus"
```bash
openclaw config.patch '{"ai": {"model": "<МОДЕЛЬ>"}}'
```
Відповідь: `✓ Переключено на <назва>. Активна з наступного повідомлення.`

Моделі:
- `haiku` → `claude-haiku-4-5-20251001`
- `sonnet` → `claude-sonnet-4-6`
- `opus` → `claude-opus-4-6`

## SSH патчинг (стандарт Сашка)

### Невеликі зміни
```bash
sed -i 's/old/new/g' file.py
```

### Великі блоки
```bash
cat > /tmp/patch.py << 'EOF'
import re
# ... patch logic ...
# Всередині triple-quoted strings пиши \\n замість \n
EOF
python3 /tmp/patch.py
```

### Заборонено
- **Base64** для передачі коду через SSH (SyntaxError з `\n` та Cyrillic)
- **scp** для передачі файлів
- **nano** для `.md` файлів (з'їдає `#` і нумерацію при PuTTY paste)
- Для `.md` використовувати `cat > /tmp/patch.py << 'PYEOF'` + triple-quoted string + `python3 /tmp/patch.py`

## 💰 Cost awareness

### Попереджай перед:
- Операціями >$0.50
- >6-8 tool calls
- Read файлів >50KB

Формат:
```
💰 УВАГА: ~$X, ~N tool calls
[що буду робити]
Продовжити?
```

### Session cost estimator (при старті)
Після привітання показати:
```
💰 Контекст: ~Nk токенів (~$X за повідомлення на [модель])
```

Формула (приблизна):
- Haiku: $0.001 за 1k input
- Sonnet: $0.003 за 1k input
- Opus: $0.015 за 1k input
